Node.js

First of all, we need to set up the environment. Angular is built, and runs on top of, Node.Js and this will have to be installed onto your system before continuing. Details on how to install this can be found at nodejs.org or by clicking on the image below.

Node.js is a server side version of Javascript, built using Chrome's Javascript engine. Unlike traditional Javascript, this framework allows for server side applications like REST APIs to be created within the one language.


Once this is installed, you'll need to use the node package manager (NPM) from your command line to install Angular. To do this, simply type:

npm install -g @angular/cli

This may take a little while depending on your internet, but once the installation is complete, you're finished setting up! Now to create your first app

Your First App

So you now have Angular installed, and creating your first app couldn't be easier. The first thing you'll need to do is navigate within your command prompt to where you would like to create your new Angular application. Once you have done this, you will need to type the following command:

ng new first-app

This command will generate the framework for a basic Angular application for you to develop in any way you choose. Although you could jump straight into trying to edit the application from here, why don't we start with a brief look at the default application. Simply enter the following commands:

cd first-app ng serve --open

And wait. This will open up a new browser window or tab with your new application. This page may not seem like much, but it's the basis of what we will be developing later in this tutorial.

What's going on?

If we now go back to look at what was generated by the 'ng new' command, it all gets a bit confusing. There are a series of folders which each contain a jumble of subfolders and files. Fortunately we can ignore most of this for now, and move to the 'src/app' folder. This is the most important part of the Angular app for us to understand as this is where the majority of development will be taking place.

Inside the app folder, all we currently have is what is known as the root module ('app.module.ts') and a root component (built up of 4 files, all beginning "app.component"). If you're just looking to build a single page application, you could achieve that with just these files.

When the files were generated you should've ended up with an app.component.ts file that looks like this:

app.component.ts

All this code does (for now) is dictate that this is a component, the identifier of the component, and where to find the markup and styling for it. These files are generated automatically for you when you create a new component, by navigating to your app root in a CLI and typing:

ng generate component *

Tip: Replace the * with the name of the component you would like to generate - for example 'Home'

This allows you to split each component, or page, from one another and creating an easy to follow structure. However, you will notice that although you have generated your files, you cannot actually see any new pages in your application. This is where the Angular router is needed - which we will implement in the next section.
